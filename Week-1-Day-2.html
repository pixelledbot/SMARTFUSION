  • Architecture of IoT

    Diagram:

    Key Features of IoT Architecture:

    • Acts as a reference model for applying IoT in businesses and services.
    • Uses smart sensors to capture data, perform preprocessing, and send it to communication managers.
    • Sensors connect to a gateway that performs data gathering, computation, and transformation.
    • The communication subsystem contains protocol handlers, message routers, and caching systems.
    • Manages device identity and access control.
    • Data is routed via the internet to enterprise application servers.
    • Analysis subsystems enable integration with business processes and enterprise systems.
  • Sensors and Actuators

    Sensors: Devices that detect physical inputs (temperature, light, pressure, motion) and convert them into signals readable by electronic systems.

    Types of Sensors: Temperature sensors (LM35), IR sensors, Ultrasonic sensors, Gas sensors, etc.

    Actuators: Devices that convert control signals into physical action (movement, switching).

    Examples: Motors, LEDs, buzzers, relays.

    Sensors sense the environment, actuators act on it — both are fundamental to any embedded or IoT system.

  • Introduction to NodeMCU and Raspberry Pi

    NodeMCU: An open-source IoT platform based on the ESP8266 WiFi module. It has digital/analog I/O pins, built-in WiFi, and is programmable via the Arduino IDE using Lua or C/C++.

    Raspberry Pi: A mini single-board computer that can run Linux OS. It supports USB, HDMI, camera, GPIO, and networking, making it powerful for real-time applications and server-based projects.

    Key Difference: NodeMCU is for sensor-level embedded applications, Raspberry Pi is for more advanced OS-level computation and interfacing.

  • Wokwi Simulator

    Diagram:

    Wokwi: A web-based simulator that allows you to simulate Arduino, ESP32, NodeMCU, and Raspberry Pi Pico projects online.

    It supports drag-and-drop components, real-time code editing, serial monitor, and sensor emulation. Ideal for beginners and testing circuits without physical hardware.

  • Arduino IDE Installation and Working

    Installation: Download the Arduino IDE from the official website (arduino.cc). Follow OS-specific instructions for Windows, macOS, or Linux.

    Working:

    • Connect your Arduino board via USB.
    • Select the board and port from the Tools menu.
    • Write or load a sketch (program) and click Upload.
    • The program is compiled and sent to the board's memory.
    • Use Serial Monitor to debug and view sensor outputs.
  • Difference between Arduino, NodeMCU and Raspberry Pi
    Feature Arduino NodeMCU Raspberry Pi
    Type Microcontroller board Microcontroller with WiFi Single-board computer
    Operating System None None Runs Linux OS
    Connectivity No built-in WiFi Built-in WiFi WiFi, Ethernet, Bluetooth (via adapters)
    Programming Language C/C++ (Arduino IDE) C/C++, Lua (Arduino IDE) Python, Java, C/C++
    Use Case Simple sensor-based projects IoT projects with WiFi Multimedia, servers, OS-level apps
