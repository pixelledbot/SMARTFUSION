Week 1 - Day 3

Topics Covered

  • Cloud Computing Introduction

    Cloud computing is a model for delivering computing services like storage, databases, networking, software, analytics, and intelligence over the internet, offering faster innovation, flexible resources, and economies of scale.

  • Traditional vs Distributed Computing
    AspectTraditional ComputingDistributed Computing
    StructureCentralizedMultiple nodes across networks
    ScalabilityLimitedHighly Scalable
    Failure HandlingSingle point of failureRedundant and fault-tolerant
    Use CasesPersonal apps, small-scale systemsCloud, big data, global services
  • Distributed Computing

    Distributed computing is a field of computer science that studies distributed systems. In these systems, a group of computers works together, appearing to the end-user as a single coherent system. These systems share the load and work in parallel to solve problems.

    Goals: Transparency, scalability, resource sharing, reliability.

    Challenges: Synchronization, security, fault tolerance, consistency.

  • Client-Server Architecture

    A model where client devices send requests to a server, which processes and returns results. Widely used in web services.

  • Trends in Distributed Computing

    Cluster Computing: Multiple computers work together as a single system to solve complex tasks.

    Grid Computing: Resources from various locations used for a common goal, handling large-scale computation.

    Utility Computing: Pay-per-use model where resources are rented (like electricity usage).

    Cloud Computing: Access computing resources via the internet on demand.

  • Virtualisation

    Virtualisation allows multiple operating systems to run on a single physical machine by creating virtual environments using software like hypervisors. This increases resource utilization and flexibility.

  • Cloud Definition Framework

    The Cloud Framework includes service and deployment models:

    Service Models

    • IaaS (Infrastructure as a Service): Provides virtual machines, storage, and networks (e.g., AWS EC2).
    • PaaS (Platform as a Service): Offers tools and services for application development without managing infrastructure (e.g., Google App Engine).
    • SaaS (Software as a Service): Ready-to-use applications over the internet (e.g., Gmail, Microsoft 365).

    Deployment Models

    • Public Cloud: Services available to anyone over the internet.
    • Private Cloud: Services maintained on a private network.
    • Hybrid Cloud: Combines public and private clouds.
    • Community Cloud: Shared among organizations with similar needs.
  • Cloud for IoT

    Cloud platforms allow IoT devices to connect, send data, and receive commands. It provides storage, analytics, and visualization tools.

    Management: Cloud provides tools to manage devices, update firmware, monitor activity, and set security rules.

    IIoT Platforms:

    • Remote monitoring and diagnostics
    • Real-time analytics
    • Interoperability with industrial protocols
    • Examples: AWS IoT Core, Azure IoT Hub, Google Cloud IoT
  • Cloud Providers Comparison
    Provider Primary Language Storage Popular Services
    Google App Engine Python, Java, Go Cloud Storage App Hosting, Firebase
    Microsoft Azure .NET, Java, Python Azure Blob VMs, Azure Functions
    AWS Java, Python, Node.js S3 EC2, Lambda, DynamoDB
  • ThingSpeak & IoT Apps

    ThingSpeak: A cloud-based platform for collecting and analyzing sensor data in real time. It supports MATLAB integration and is ideal for prototyping IoT applications.

    Example IoT Applications:

    • Smart Homes
    • Smart Farming
    • Health Monitoring
    • Traffic Management
